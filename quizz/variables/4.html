Explication En mode strict, il est interdit de lire une variable qui n'est pas déclarée.
Dans une fonction les variables sont déclarées au tout début de l'exécution de la fonction quel que soit l'endroit où cette déclaration est écrite dans le code. C'est ce qu'on appelle le "hoisting". Ainsi la variable test est bien déclarée dans la fonction. Le mode strict ne générera donc pas d'erreur. Cependant l'affectation de cette variable n'est réalisée que lorsque l'interpéteur rencontre la ligne en question. Donc test vaut toujours undefined au moment de l'affichage.
