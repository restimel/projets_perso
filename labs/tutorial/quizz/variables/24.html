Explication En JavaScript, les variables sont créées avant le début de l'exécution de la fonction, quel que soit l'endroit où elles sont déclarées dans le code (c'est ce qu'on appelle le "hoisting").
Cela signifie qu'il est possible d'appeler une fonction qui est déclarée plus loin dans le code. Et même si celle-ci est déclarée après un return qui met fin à l'exécution du code dans une fonction.

Quand la fonction f est appelée, elle retourne une référence à la fonction g. Donc h possède une référence à la fonction g. Cette fonction n'a pas été appelée donc typeof h vaut "function".
